# 5.5. 小结：单机 vs 远程

到目前为止，你已经掌握了 Git 的核心操作——而且所有这些都不需要网络连接。

## 你已经学会的本地操作

回顾一下，以下命令全部可以在离线状态下完成：

- `git init` — 创建本地仓库
- `git add` — 暂存更改
- `git commit` — 保存提交
- `git log` — 查看历史
- `git branch` — 创建和列出分支
- `git switch` — 切换分支
- `git merge` — 合并分支
- 解决合并冲突

**这些操作已经提供了版本控制系统的核心功能。** 就像 SVN 一样，你可以提交代码、查看历史、创建分支、合并代码——所有这些都只需要在你的电脑上完成，不需要网络。

Git 首先是本地工具，网络功能是后续才会用到的扩展。

## 远程操作只是延伸

接下来你会学习仓库之间的操作。很容易想象，远程操作无非是两件事：下载与上传。

- 把你的本地库发往远程：`git push`
- 把远程库更新到本地：`git pull`

没了。

当然，在能执行这两步之前，你还得做一些配置，否则本地库怎么知道对应的远程库是哪里？下一章就会讲这些。
