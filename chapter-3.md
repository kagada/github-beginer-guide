# 第3章：撤销和修正：犯错也没关系

上一章，你学会了如何保存修改和查看历史。但人总会犯错——提交了错误的文件、写了糟糕的提交信息、或者改乱了代码想重来。

别担心，Git 提供了多种方式来撤销操作。记住：**在本地，几乎任何错误都可以挽回**。

## 看看你改了什么

在提交之前，你可能想先检查一下自己到底改了什么。打开 `notes.txt`，随便加几行内容，然后运行：

```bash
git diff
```

你会看到类似这样的输出：

```diff
diff --git a/notes.txt b/notes.txt
--- a/notes.txt
+++ b/notes.txt
@@ -1 +1,3 @@
 这是我的学习笔记
+
+今天学了 Git 的撤销功能，很有用！
```

绿色的 `+` 行是你新增的内容，红色的 `-` 行是你删除的内容。

`git diff` 让你清楚地看到：自上次提交以来，你对文件做了哪些具体修改。这有助于你在提交前确认变更是否正确。

## 取消暂存：放错了"购物车"怎么办

假设你不小心把 `notes.txt` 加入了暂存区，但现在还不想提交它：

```bash
git add notes.txt
```

别慌，用这个命令把它从暂存区拿出来：

```bash
git restore --staged notes.txt
```

运行 `git status` 看看——`notes.txt` 仍然在你的工作目录里，但已经不在暂存区了。你的修改没有丢失，只是撤销了 `git add` 的效果。

**`git restore --staged` 只是把文件移出暂存区，不会丢弃你的修改。**

## 丢弃修改：我想回到上次提交的状态

有时候，你改乱了文件，想完全放弃这些修改，回到上次提交时的样子。

**警告：这个操作会永久丢失未提交的修改，无法恢复。确认你真的不想要这些修改后再执行。**

```bash
git restore notes.txt
```

运行后，`notes.txt` 会恢复到上次提交时的内容。你在编辑器里做的所有修改都会消失。

**区别总结**：
- `git restore --staged <file>` — 移出暂存区，保留修改
- `git restore <file>` — 丢弃工作区的修改，回到上次提交

## 修正最后一次提交

你刚提交了一个 commit，然后发现：
- 提交信息写错了
- 忘记把某个文件加进去

这时候不需要创建一个新的提交来修正，Git 允许你"改写"最后一次提交：

```bash
git commit --amend -m "添加学习笔记和 Git 笔记"
```

这个命令会用新的提交信息替换最后一次的提交信息。

如果你想把忘记的文件加进最后一次提交：

```bash
git add forgotten-file.txt
git commit --amend --no-edit
```

`--no-edit` 表示保持原来的提交信息不变，只是把新文件加入最后一次提交。

**`git commit --amend` 会创建一个新的提交替换最后一次提交**，而不是在原地修改。旧的提交会被丢弃（只要还没推送到远程，这就完全安全）。

## 犯错是正常的

学习 Git 的过程中，你会犯错。这完全正常。

关键是要记住：
- **本地操作很安全** — 只要还没推送到远程，几乎所有错误都可以修正
- **先查看，再提交** — 用 `git diff` 和 `git status` 确认变更
- **别怕试错** — 最好的学习方式就是动手实践，反正可以撤销

## 你刚刚学会了什么

- `git diff` — 查看自上次提交以来的修改
- `git restore --staged <file>` — 把文件移出暂存区，保留修改
- `git restore <file>` — 丢弃未提交的修改
- `git commit --amend` — 修正最后一次提交
