# 5.5. 小结：单机 vs 远程

到目前为止，你已经学完了 Git 几乎所有的核心操作。

- 创建仓库、保存提交：`init`、`add`、`commit`
- 查看历史、比较变更：`log`、`diff`
- 撤销和修正：`restore`、`commit --amend`
- 创建分支、合并分支、解决冲突：`branch`、`switch`、`merge`

**这些操作全部可以在离线状态下完成。** 就像 SVN 这类版本控制系统一样，提交代码、查看历史、创建分支、合并代码——这些版本控制的核心功能，Git 不需要联网就能做到。

## 那远程操作呢

接下来你会进入新的层次：仓库与仓库之间的操作。

远程操作无非就是两件事：

- 把本地提交发到远程：`git push`
- 把远程提交拉到本地：`git pull`

没了。

当然，push 和 pull 之前还得做一些配置，让本地库知道远程库在哪里。下一章就开始讲这些。
