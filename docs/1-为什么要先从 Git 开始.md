# 1. 为什么要先从 Git 开始？

如果你刚开始接触编程，可能会听到"把代码放到 GitHub 上"这样的话。于是你很自然地认为，学习 GitHub 就是第一步。

但我们要说的是：**不，先学 Git。**

## Git 和 GitHub 是两回事

这就像"引擎"和"汽车品牌"的关系。

- **Git** 是一个版本控制工具，由 Linux 之父 Linus Torvalds 在 2005 年创建。它的核心任务是帮你追踪文件的修改历史，让你可以随时回到过去的任何一个版本。
- **GitHub** 是一个网站平台，由其他人在 2008 年创建。它使用 Git 作为底层技术，提供了代码托管、协作、讨论等功能。

一个简单的事实：**是 GitHub 需要 Git，而不是 Git 需要 GitHub。** 在 GitHub 出现之前的 3 年里，Git 已经被全世界的开发者广泛使用。GitHub 只是众多基于 Git 的平台之一——还有 GitLab、Bitbucket、Gitee 等。

学会 Git 的核心操作，就能快速上手各种 Git 平台。

## 为什么从本地开始？

直接上手 GitHub 会让新手面临两个问题：

1. **操作黑箱感**：你在 GitHub 上点了一个按钮，做了一步操作，但你不清楚它背后发生了什么。万一做错了呢？怎么撤回？对很多新手来说，GitHub 像个黑箱，一旦发生什么，就难以挽回
2. **概念过载**：Git、GitHub、远程仓库、SSH 密钥、Pull Request……太多概念堆在一起，哪些要用、哪些跟你的需求无关，很难厘清

**本地 Git 操作完全不同**：
- 所有命令都在你的终端里运行，你能看到每一步发生了什么
- 所有操作都可以撤销，第4章会专门教你"搞砸了怎么办"
- 你可以专注于理解版本控制的核心概念，不用同时面对网络和协作的复杂性

等你在本地熟练了，再连接到 GitHub，那些"黑箱"操作就不再神秘了。
